##  ПОСТАНОВКА ЗАДАЧИ (из PDF)

| Параметр | Описание |
|---------|--------|
| **Колонки** | До 1000, высота до 26 |
| **Птицы** | A, B, C… (заглавные латинские) |
| **Правила перемещения** | Только **крайняя** птица → **пустая колонка** или **на ту же птицу** |
| **Цель** | Все колонки: **полностью заполнены одним видом** или **пустые** |
| **Целевая функция** | `F = 100·N·L − K`  
| → `N` — высота колонок  
| → `L` — число **полностью собранных** колонок  
| → `K` — число **перемещений** |

## ФОРМАТ ВХОДНЫХ ДАННЫХ

```text
DATA
A B C   ← колонка 1 (снизу вверх)
B A C   ← колонка 2
...
==      ← пустая колонка
== 
/
```

- **Птицы в строке — снизу вверх** (последняя в строке — **крайняя**)
- **Все непустые колонки — одинаковой длины `N`**
- **Число птиц каждого вида — кратно `N`**

## АЛГОРИТМ: **A* + BEAM SEARCH**

A* — это "умный" поиск пути, который комбинирует:
g(x) — цена уже сделанных шагов (количество перемещений);
h(x) — эвристика, оценка, насколько далеко мы от цели;
f(x) = g(x) + h(x) — приоритет, чем меньше — тем лучше.
Он выбирает, какие состояния исследовать дальше, опираясь на минимальный f(x).

Beam Search — это ограничение количества рассматриваемых вариантов.
